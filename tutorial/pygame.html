Sonoplastia (Mixer) e Tratamento de Erros

O módulo pygame.mixer lida com o áudio. Usamos duas ferramentas distintas:

  • mixer.music: Focado em tocar arquivos .mp3 longos em loop (trilha sonora), poupando memória RAM.
  • mixer.Sound: Focado em arquivos .wav curtos, ideais para efeitos sonoros (tiros e portas) que precisam ser processados sem atraso.

Usamos blocos de try...except (Tratamento de Exceções nativo do Python) para carregar os sons. Se o usuário esquecer de baixar o arquivo .mp3, o bloco except impede que o jogo sofra um crash (fechamento abrupto) e permite jogar sem áudio.

Interfaces Transparentes (Overlay)

Para criar a tela de Vitória/Derrota que escurece a fase no fundo, criamos uma Surface no Pygame ativando a flag pygame.SRCALPHA.

Isso permite que a cor preta (0, 0, 0) receba um quarto valor (o canal Alpha), ex: (0, 0, 0, 180). Assim, o Pygame desenha um retângulo preto com 180 níveis de opacidade (sendo 255 totalmente sólido), criando o clássico efeito de vidro escurecido de menu.
